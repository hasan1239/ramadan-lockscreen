You are extracting prayer times from a mosque timetable image into a standardised JSON format. Your goal is 100% accuracy — every time value must exactly match what is printed on the timetable.

This prompt works for ANY mosque timetable — any month, any location, any format. Timetables may be for Ramadan (which includes fasting-related columns like Sehri Ends and Iftari) or for any other month of the year (which typically omit fasting columns). Both formats are fully supported.

STEP 1: SURVEY THE ENTIRE IMAGE
Before extracting any data, scan the FULL image including:
- The main table with rows and columns
- Column headers (which may be in Arabic, English, or both)
- Vertical/rotated text running alongside columns (commonly used for Fajr Jama'at rules, Zuhr Jama'at times, and Maghrib Jama'at rules)
- Footnotes, bottom text, and side notes (these frequently contain fixed jama'at times, Jumu'ah times, Eid salah times, Sadaqatul Fitr amounts, and radio frequencies)
- The mosque name, address, and contact details (usually at the top or bottom)
- The month, year, and Islamic month/year (e.g. "Ramadan 1447 AH", "June 2025", "Dhul Hijjah 1446")

IMPORTANT: Some timetables use RIGHT-TO-LEFT column ordering (date/day on the far right, prayer times reading right-to-left). Do NOT assume columns read left-to-right. Always read the column HEADERS first and match each header to its data column, regardless of direction. If you see dates on the right side and Isha on the left side, the table is right-to-left.

STEP 2: IDENTIFY COLUMN MAPPING
Timetables vary widely in column names. Map each column you see to the correct output field:

  Sehri Ends / End of Suhoor / Suhur End / End of Sehri / Sehri → "sehri_ends" (Ramadan only — non-Ramadan timetables typically omit this column)
  Fajr Start / Fajr Begins / Fajr Beginning Time / Subha Sadiq / Subah Sadiq / Fajr / Dawn → "fajr_start" (outside Ramadan this is often the primary pre-dawn column; during Ramadan it may coexist with Sehri Ends)
  Sunrise / Sun-Rise → "sunrise"
  Zawal / Zawāl / Zenith / After Zenith → "zawal"
  Zohr / Zuhr / Dhuhr Start / Dhuhr Begins / Zuhr Start / Zuhr Begins → "zohr"
  Asr Start / Asr Begins → "asr"
  Esha / Isha Start / Isha Begins → "esha"
  Fajr Jama'at / Fajr Jamaat / Fajar Jamaat → "fajr_jamaat"
  Zohar Jama'at / Zuhr Jama'at / Dhuhr Jama'at / Dhuhr Jamaat / Zuhr Jamaat / Zohr & Juma → "zohar_jamaat"
  Asr Jama'at / Asr Jamaat → "asr_jamaat"
  Maghrib / Maghrib Start / Maghrib Begins / Sunset / Iftaar / Iftar / Iftar Time / Maghrib Azan / Iftaar Maghrib / Maghrib Azan & Iftari → "maghrib_iftari" (during Ramadan this is the Iftar time; outside Ramadan it is simply the Maghrib prayer start time — use the same field regardless)
  Maghrib Jama'at / Maghrib Jamaat → "maghrib_jamaat"
  Esha Jama'at / Isha Jama'at / Isha Jamaat / Taraweeh Jamaat → "esha_jamaat" (Taraweeh is Ramadan only — outside Ramadan this is just regular Esha Jama'at)

IMPORTANT COLUMN DISAMBIGUATION:

Sunset vs Maghrib/Iftar: Some Ramadan timetables have BOTH a "Sunset" column AND a separate "Maghrib" or "Iftar" column. These are different values — Sunset is the astronomical event, while Maghrib/Iftar is a few minutes later when the prayer begins and the fast is broken. If both columns exist, ALWAYS use the Maghrib/Iftar column for "maghrib_iftari", NOT the Sunset column. If only one of these columns exists (common outside Ramadan where there is just "Maghrib" or "Sunset"), use whichever is available for "maghrib_iftari". CRITICAL: The Sunset column is NOT Isha/Esha Start. Sunset values will be very close to Maghrib (within a few minutes), while Isha Start is always significantly later (typically 1-2 hours after Maghrib). Do NOT map Sunset values to "esha". If both Sunset and a separate Maghrib column exist, ignore the Sunset column.

Maghrib Start vs Maghrib Jama'at: Some timetables have BOTH a Maghrib start time column (labelled "Maghrib", "Iftar Time", "Sunset", etc.) AND a separate "Maghrib Jama'at" column. These are different — the start time is when the Maghrib prayer window begins (and during Ramadan, when the fast breaks), while Maghrib Jama'at is when the congregation prays (a few minutes later). Map the start time to "maghrib_iftari" and Maghrib Jama'at to "maghrib_jamaat". Do NOT confuse them — they are separate fields. Note: Some timetables label a column "Maghrib (Iftar)" even under a "Jama'at Times" header — this is still the start/iftar time, not the Maghrib Jama'at. The Maghrib Jama'at may be specified separately in a footnote (e.g. "Maghrib Jama'at 6 minutes after Azan").

Sehri Ends vs Fajr Start: Ramadan timetables may have BOTH a "Sehri Ends" / "End of Suhoor" column AND a separate "Fajr Start" / "Fajr Begins" / "Subah Sadiq" column. These are different — Sehri Ends is when eating must stop, while Fajr Start / Subah Sadiq is when the Fajr prayer window begins (a few minutes later). Map Sehri Ends to "sehri_ends" and Fajr Start / Subah Sadiq to "fajr_start". Do NOT confuse them — they are separate fields. If only one pre-dawn column exists: during Ramadan (if the timetable mentions Sehri/Suhoor), map it to "sehri_ends" and leave "fajr_start" as ""; outside Ramadan (if the column is labelled "Fajr", "Fajr Start", "Dawn", "Subah Sadiq", etc. with no mention of Sehri/Suhoor), map it to "fajr_start" and leave "sehri_ends" as "". IMPORTANT: Some timetables combine both labels into a single column header (e.g. "Dawn / Subh Sadiq / Suhoor Ends" or "Sehri End / Fajr Start"). When a SINGLE column has BOTH labels, it is one column with one set of values — map it to "sehri_ends" and leave "fajr_start" as "". Only populate "fajr_start" when there are TWO separate columns with their own distinct values. Be especially careful not to confuse "Fajr Start" with "Fajr Jama'at" — they are adjacent columns with different values. Fajr Start is when the prayer window opens; Fajr Jama'at is when the congregation prays (later).

Zawal/Zenith vs Zohr/Zuhr: Some timetables have BOTH a "Zawal" / "Zenith" / "After Zenith" column AND a separate "Zohr" / "Zuhr" / "Dhuhr Start" column. These are different — Zawal is when the sun passes its zenith (and prayer is NOT permitted), while Zohr/Zuhr is when the Dhuhr prayer window begins (a few minutes later). Map Zawal to "zawal" and Zohr/Dhuhr Start to "zohr". Do NOT confuse them — they are separate fields. If only one of these columns exists, determine which it is from context: if the value is a few minutes before other Dhuhr-related times, it is likely Zawal; if it matches or is close to the Dhuhr Jama'at time, it is likely Zohr. When only one exists, map it to "zohr" and leave "zawal" as "".

NOTE ON PRE-MONTH ROWS:
Some timetables include rows before the main month begins (e.g. a Ramadan timetable starting from the day before Ramadan). Include ALL rows shown in the timetable — do not skip or renumber them.

NOTE ON JAMA'AT-ONLY TIMETABLES:
Some timetables only show Jama'at (congregational) times and do not include prayer start times at all. For example, a timetable might show Sehri End, Fajr Start, Sunrise, and then only Jama'at columns for Fajr, Zuhr, Asr, Maghrib, and Isha — with no Zuhr Start, Asr Start, or Isha Start columns. This is normal. Map what exists and leave the missing start time fields as "". Do NOT invent start times that aren't shown.

STEP 3: HANDLE SPECIAL PATTERNS

DITTO MARKS — CRITICAL:
Many timetables use ditto marks to mean "same as the row above". These appear as:
  "  (double quote)
  ''  (two single quotes)
  ""  (two double quotes)
  «  (guillemet)
  ″  (double prime)
  Empty/blank cells that clearly should have a value
You MUST resolve ALL of these to the actual repeated value. Walk down each column: if a cell has a ditto mark, copy the value from the nearest non-ditto row above it. NEVER output a ditto mark — every cell must have an explicit time value or be genuinely empty. Pay special attention to Jama'at columns (especially Zuhr Jama'at) where a value like "1:30" may appear on certain rows (often Fridays) with ditto marks or blank cells on other rows — these blank cells mean the same time applies to ALL rows, not just the rows with explicit values. If a Jama'at column has some explicit values and the rest are blank/ditto, resolve them all.

VERTICAL/ROTATED TEXT:
Some columns have a rule written vertically instead of individual values per row. Common examples:
  - "FIFTEEN MINUTES AFTER SEHRI ENDS" / "Fajr will be 15 minutes after suhur end" / "TEN MINUTES AFTER SEHRI ENDS" / "15 minutes after Fajr start" → for fajr_jamaat, calculate the actual time for each row by adding the stated minutes to the value the rule references. Pay close attention to what the rule says: if it says "after Sehri ends" add to the sehri_ends value; if it says "after Fajr start" or "after Fajr beginning time" add to the Fajr Start column (even if that is a different column from sehri_ends). Output the calculated time, not a formula (e.g. Sehri 5:42 + 15 mins → fajr_jamaat "5:57"). HOWEVER: if the Fajr Jama'at column also contains explicit per-row time values, ALWAYS use the explicit values from the table — they take priority over any rule or note. The rule may be approximate guidance while the table values are the mosque's actual scheduled times.
  - "Zuhar Jamat 1:00" / "Jumu'ah Jamat 1:00" → set zohar_jamaat to "1:00" for all rows (or apply Jumu'ah-specific time on Fridays if different). ALSO extract any Jumu'ah time mentioned here into the "jummah_times" metadata field — do not just put it in "notes".
  - "Maghrib Jama'at, 10 minutes after Azan" / "5 minutes after Iftar" / "5 minutes after Maghrib" / "straight after breaking fast" / "straight after Azan" → for maghrib_jamaat, if the timetable has explicit per-row values, use those. If the rule is written as vertical text with no per-row values, calculate the actual time for each row (e.g. Maghrib 5:31 + 5 mins → maghrib_jamaat "5:36"). If the rule says "straight after" or similar with no specific minutes, set maghrib_jamaat to "" and note the rule in "notes" instead.

HEADER NOTES AND ANNOUNCEMENTS:
Some timetables show key information in large text ABOVE the table (not in footnotes). For example:
  - "ZOHR 1:00PM & JUMUAH @ 1:15PM" → extract into jummah_times metadata (e.g. "1:15pm") AND apply: zohar_jamaat = "1:00" on non-Fridays, "1:15" on Fridays.
  - "Fajr Jamaat will be 15 mins after Sehri Ends" → apply the rule (but explicit table values take priority if present).
  Treat these header notes the same as footnotes — extract relevant data into the correct metadata fields.

FOOTNOTES AND BOTTOM NOTES:
Look carefully at text below or beside the table. Common patterns:
  - "Daily Zuhr Jama'at: 1:00pm" / "Zuhr Jamaat - 12:40pm" → apply this time to zohar_jamaat for all non-Friday rows
  - "Jumu'ah at 12:30pm & 1:40pm" / "First Jamaat @ 12:45pm Second Jamaat @ 1:30pm" → extract into jummah_times metadata AND use the FIRST jumu'ah time as zohar_jamaat on Fridays
  - "Eid Salah: 7:30am & 9:00am" → extract into eid_salah metadata
  - "Sadaqatul Fitr: £7" / "Sadq-E-Fitr £5.00" / "Fitrana £6 per person" → extract into sadaqatul_fitr metadata
  - "Radio Frequency: 454.3500" / "Freq: 461.2375" → extract into radio_frequency metadata

FRIDAY OVERRIDES FOR ZOHAR JAMA'AT:
On Fridays, the Zuhr/Dhuhr Jama'at time often changes to a Jumu'ah time. Check:
1. Does the table show explicit different values on Friday rows? → Use those values.
2. Does a footnote specify a Jumu'ah time different from the daily Zuhr Jama'at? → Use the first Jumu'ah time on Fridays, the daily time on other days.
3. If no distinction is made, use the same time for all days.

DATE FORMATTING:
Some timetables show full dates ("18 Feb"), others show only day numbers ("18", "19", "20"...) without the month. If only day numbers are shown, you must infer the month from context: look at the timetable title, headers, or footnotes for the month/year information, and combine it with the day numbers. If the timetable spans two months (e.g. February into March), determine where the month changes by looking at when day numbers reset from a high number back to 1 (e.g. 28 → 1 means Feb → Mar). Always output dates in "DD Mon" format (e.g. "18 Feb", "1 Mar").
